# 用户管理重构计划

## 当前问题分析

- 现有用户管理页只有单张表单卡片，信息层级过于扁平，用户只能感知“改资料”，看不到“账户概览”和“设置”这两个更长期的管理入口。
- 退出登录按钮直接作为侧边栏底部的高强调危险按钮展示，视觉权重过高，容易干扰主导航，也会让常用操作和低频破坏性操作混在一起。
- 前端没有为“用户设置”预留固定入口，后续如果增加通知偏好、界面偏好、隐私与安全等能力，需要重新调整导航和页面结构。

## 本次重构

- 侧边栏底部从“用户信息 + 独立退出按钮”调整为“用户资料卡 + 账户菜单”。
- 将退出登录收敛到账户菜单内，降低视觉干扰，同时补充“个人资料”和“用户设置”入口。
- 重构用户页布局，拆分为“账户头部信息”“基础信息编辑”“账户概览”“用户设置（预留）”四个信息块。
- 在用户页中显式预留 `#settings` 区域，先固定信息架构，避免后续新增设置能力时再次打断导航设计。

## 后续执行计划

1. 新增独立的 `/settings` 页面，将当前预留区迁移为真正的设置中心，并按“通知偏好 / 界面偏好 / 隐私与安全”拆分分组。
2. 为用户偏好建立前端状态模型和本地缓存策略，明确哪些设置仅前端生效，哪些需要和后端持久化。
3. 与后端补齐用户设置接口定义，优先覆盖默认首页、消息提醒、登录安全策略三个高频项。
4. 将用户页拆分为更小的可复用组件，补齐交互测试，避免后续继续把账户功能堆回单页里。
